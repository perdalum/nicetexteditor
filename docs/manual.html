NiceTextEditor Manual

A small plain-text editor with a worksheet soul.

NiceTextEditor is a native macOS editor for ordinary text files. It keeps documents plain, but gives each window comfortable reading controls, color, line numbers, standard macOS text assistance, and MPW/BBEdit-style shell worksheets.

1. Overview

NiceTextEditor opens and saves regular plain-text files. It is built from SwiftUI document windows with an AppKit NSTextView editing surface, so it feels like a native macOS text editor rather than a web view or custom canvas.

The guiding idea is simple: the file stays plain. Visual preferences such as zoom, tab width, colors, and line numbers are per-window conveniences. They are not written into the document.

Plain-text promise: worksheet output can insert text because you asked it to, but editor UI state is not stored in the file.

2. Documents and windows

Use the standard macOS document commands to create, open, save, duplicate, rename, and close files. Each open document has its own editor window, status bar, toolbar choices, and worksheet shell process.

  • New: creates an untitled plain-text document.
  • Open: opens existing plain-text files.
  • Save / Save As: writes normal text content only.
  • Multiple documents: each window is independent.

The status bar shows the document name, file location when available, and character count. On narrow windows it automatically simplifies itself so the editor remains usable.

3. Toolbar controls

The toolbar holds per-window visual controls. These are useful for adjusting one document without changing another.

Background

Choose the editor background for this window. Options include System, soft pastels, Black, and White. Pastel choices include dark-mode counterparts.

Text

Choose the foreground color for both proportional text and monospace verbatim text. Amber and Green are included for a retro terminal feel.

Lines

Toggle Xcode-style line numbers in a left gutter. Line numbers are logical document lines, not wrapped visual rows.

Text size

Choose a zoom preset or use Zoom In, Zoom Out, and Actual Size. Zoom is local to the window.

Tab width

Choose visual tab width presets: 2, 4, 8, or 16. Tabs remain tab characters in the file.

Full-screen width

Choose how wide the text should be in full screen. This makes full-screen editing more readable on wide displays.

4. Editing text

NiceTextEditor uses standard macOS text editing behavior: selection, copy, paste, undo, redo, drag selection, contextual menus, and services work through AppKit.

Spelling, grammar, and substitutions

The editor enables the standard macOS text assistance system. Look in the Edit menu for:

  • Spelling and Grammar
  • Substitutions
  • Smart quotes and smart dashes
  • Text Replacement
  • Link detection and data detectors

These features can change typed text when enabled, just like in other native macOS text views.

5. Find and Go To Line

Find

Use standard macOS find commands:

  • ⌘F opens the find interface.
  • ⌘G moves to the next match.
  • ⇧⌘G moves to the previous match.

Go To Line

Use Edit > Go To Line… or ⌘L. The prompt accepts line numbers and aliases.

InputMeaning
i, where i is within the fileGo to line i.
i greater than the number of linesGo to the last line.
0Go to line 1.
start or sGo to line 1.
end, eof, or eGo to the last line.
negative numberCount back from the end, clamped to the document.

6. UNIX worksheet

Each document owns its own background /bin/zsh process. Terminal is not launched. The shell belongs to the document window, so separate documents can maintain separate shell state.

% date
Tue Jun 16 21:00:00 CEST 2026

Run selected commands

Select text containing a shell command and press ⇧Return, or use Worksheet > Run Selection in Shell. Output is inserted after the selection.

Filter selected text

Select text, then use one of the pipeline commands:

  • ⌘E: prompt for a zsh pipeline, send the selection as standard input, and replace the selection with output.
  • ⇧⌘E: prompt for a zsh pipeline, send the selection as standard input, and insert output after the selection.

The pipeline prompt focuses the command field automatically. Use Up and Down Arrow in the prompt to browse command history shared across documents.

Startup file

NiceTextEditor creates and sources this file before starting each document shell:

~/Library/Application Support/NiceTextEditor/WorksheetStartup.zsh

Use it for PATH, aliases, shell functions, exports, and zsh options. Settings includes a button to reveal it in Finder. Use Worksheet > Reset Document Shell to restart the active shell after changes.

7. Settings

Settings contains global configuration that should apply across document windows.

  • Editor Font: choose the proportional editor font with the standard macOS Font panel.
  • Worksheet shortcuts: configure command shortcuts for worksheet execution and filtering.
  • Worksheet startup file: reveal the shell startup file in Finder.

Per-window choices such as zoom, line numbers, tab width, and colors are intentionally not stored in Settings.

8. Verbatim blocks

NiceTextEditor recognizes nroff-style .VB and .VE marker lines for display only. Text between the markers is shown in SF Mono, while the marker lines and all document content remain plain text.

This is proportional text.
.VB
This line is shown in SF Mono.
    Indentation is preserved visually.
.VE
Back to proportional text.

This is useful for code snippets, command output, and examples in otherwise prose-like text.

10. Tips

  • Use line numbers and colors to distinguish open documents during focused work.
  • Use Amber or Green text on a dark background for an old-school terminal worksheet mood.
  • Use full-screen text width presets to keep long prose readable on wide monitors.
  • If a command behaves differently from Terminal, check WorksheetStartup.zsh; the app shell is real zsh, but it has its own startup file.
  • For commands that wait for standard input forever, redirect input explicitly, for example < /dev/null.
